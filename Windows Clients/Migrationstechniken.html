Side-by-Side Migration

Beschreibung: Bei der Side-by-Side Migration werden Daten von einem alten System auf ein neues System übertragen. Dies erfolgt durch parallelen Betrieb beider Systeme, wobei die Daten von der alten Maschine auf die neue kopiert werden.

Vorteile Nachteile
Minimale Ausfallzeit: Das neue System kann vorbereitet werden, während das alte noch in Betrieb ist. Höhere Kosten: Benötigt zwei Systeme gleichzeitig, was die Hardwarekosten erhöht.
Datenkonsistenz: Da beide Systeme parallel laufen, können Daten jederzeit abgeglichen und verifiziert werden. Komplexität: Erfordert detaillierte Planung und Synchronisation.
Flexibilität: Ermöglicht einen kontrollierten und schrittweisen Übergang. Potenzielle Sicherheitsrisiken: Datenübertragung zwischen zwei Systemen kann Sicherheitsrisiken mit sich bringen.

Wipe-and-Load Migration

Beschreibung: Bei der Wipe-and-Load Migration wird das bestehende System komplett gelöscht und anschließend ein neues Betriebssystem installiert. Danach werden die Daten und Anwendungen auf das neue System migriert.

Vorteile Nachteile
Saubere Installation: Alle alten Daten, Fehler und Probleme werden entfernt. Längere Ausfallzeit: Das System ist während der Neuinstallation nicht verfügbar.
Bessere Performance: Durch die Neuinstallation des Betriebssystems können Performance-Probleme behoben werden. Risiko von Datenverlust: Wenn die Datensicherung unvollständig ist, können Daten verloren gehen.
Einfachere Verwaltung: Das neue System ist von Anfang an konsistent und sauber eingerichtet. Benutzeraufwand: Benutzer müssen möglicherweise ihre persönlichen Einstellungen und Daten manuell wiederherstellen.

USMT:

Das User State Migration Tool (USMT) ist ein leistungsfähiges Migrationswerkzeug, das von Microsoft bereitgestellt wird und Teil des Windows Assessment and Deployment Kit (Windows ADK) ist. Es wird hauptsächlich in Unternehmensumgebungen verwendet, um Benutzerkonten, Betriebssystemeinstellungen und Daten von einem Windows-basierten Computer auf einen anderen zu migrieren.

Vorteile Nachteile
Flexibel und leistungsfähig, besonders für Unternehmensumgebungen geeignet. Komplex in der Einrichtung und Verwendung.
Kann Benutzerkonten, Anwendungen und Betriebssystemeinstellungen migrieren. Erfordert Administratorrechte und Kenntnisse in der Befehlszeilenverwaltung.
Unterstützt Skriptdateien für angepasste Migrationen. -

Windows Easy Transfer:

Vorteile Nachteile
Einfach zu bedienen, benutzerfreundliche Oberfläche. Unterstützt keine Migration von Anwendungen.
Geeignet für Privatanwender. Nicht mehr in Windows 10 enthalten, daher veraltet.

Microsoft Deployment Toolkit (MDT):

Vorteile Nachteile
Umfangreiche Automatisierungsmöglichkeiten für Massenmigrationen. Erfordert umfangreiche Kenntnisse in der Bereitstellung und Konfiguration.
Kann mit anderen Microsoft-Tools wie WDS und SCCM integriert werden. Komplex und zeitaufwendig in der Einrichtung.

Beispiel-Szenario: Migration von Benutzerdaten von einem Windows 10-Rechner auf einen Windows 11-Rechner mit USMT

Vorbereitung

  • Installieren des Windows ADK: ADK für für Windows 11 herunterladen und auf beiden PCs installieren

  • Netzwerkzugriff sicherstellen: Stellen Sie sicher, dass sich beide PCs im selben Netzwerk befinden

  • Erstellen von Benutzerkonten: Auf dem Zielrechner (Windows 11) müssen die entsprechenden Nutzerkonten vorhanden sein.

  • Sichern der Daten auf dem Quell-PC (Windows 10) mit Hilfe der Eingabeaufforderung (muss Administrationsrechte haben.)

    scanstate \\Server\USMTStore /i:migdocs.xml /i:migapp.xml /o /c /v:13 /l:C:\scanstate.log
    scanstate \\Server\USMTStore /i:migdocs.xml /i:migapp.xml /o /c /v:13 /l:C:\scanstate.log

    Erklärung: \\Server\USMTStore gibt den Speicherort an, an dem die gesicherten Daten gespeichert werden sollen. migdocs.xml und migapp.xml sind Standard-USMT-Skriptdateien, die bestimmen, welche Dokumente und Anwendungen migriert werden. /o überschreibt vorhandene Daten im Speicherort. /c ignoriert nicht-kritische Fehler. /v:13 gibt das Detaillierungsgrad der Protokollierung an. /l:C:\scanstate.log gibt den Pfad zur Protokolldatei an.

Wiederherstellen der Daten auf dem Zielsystem (Windows 11):

Öffne Eingabeaufforderung mit Administratorrechten auf dem neuen Windows 11-Rechner. Die Dateien können mit dem folgenden Befehl wiederhergestellt werden:

loadstate \\Server\USMTStore /i:migdocs.xml /i:migapp.xml /v:13 /l:C:\loadstate.log
loadstate \\Server\USMTStore /i:migdocs.xml /i:migapp.xml /v:13 /l:C:\loadstate.log

Erklärung: \\Server\USMTStore gibt den Speicherort der gesicherten Daten an. migdocs.xml und migapp.xml bestimmen, welche Daten wiederhergestellt werden. /v:13 gibt das Detaillierungsgrad der Protokollierung an. /l:C:\loadstate.log gibt den Pfad zur Protokolldatei an.

Nachbereitung: Überprüfen Sie, ob alle Daten und Einstellungen korrekt übertragen wurden.

Dies ist nur eine der vielen Möglichkeiten für dieses Szenario.
